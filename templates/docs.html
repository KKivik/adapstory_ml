# Метод Эндпоинт Описание Параметры Пример
1 GET /api/users Получить список пользователей
  • limit: количество пользователей (по умолчанию 10)
  • offset: смещение (по умолчанию 0)
 /api/users?limit=5&offset=10
2 POST /api/users Создать нового пользователя
  • username: имя пользователя (обязательно)
  • email: email (обязательно)
  • password: пароль (обязательно)
 { "username": "test", "email": "test@example.com", "password": "secret" }
3 GET /api/users/{id} Получить информацию о пользователе
  • id: ID пользователя (обязательно)
 /api/users/42
4 PUT /api/users/{id} Обновить информацию о пользователе
  • id: ID пользователя (обязательно)
  • username: новое имя пользователя
  • email: новый email
 { "username": "newname", "email": "new@example.com" }
5 DELETE /api/users/{id} Удалить пользователя
  • id: ID пользователя (обязательно)
 /api/users/42
Примечания
  • Все запросы к API требуют аутентификации (кроме публичных эндпоинтов)
  • Ответы возвращаются в формате JSON
  • Базовый URL: https://api.example.com/v1
